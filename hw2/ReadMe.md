## Пояснения по задачам.

1. **Развернуть односвязный список**

В решении используются указатели `curr`, `prev` и `temp`. На каждом шаге указатель текущего элемента на следующий заменяется указателем на предыдущий элемент. Проходим по списку 1 раз, последовательно обрабатывая каждый элемент, причем условием останова является отсутствие следующего элемента, последний элемент (если он есть) обрабатываем отдельно. Поэтому решение работает за $O(n)$.

2. **Найти середину списка**

С каждым шагом расстояние между быстрым и медленным указателями увеличивается на 1. Быстрый указатель сделает n/2 (n - количество элементов в списке) шагов к моменту, когда он дойдет до конца списка. Это означает, что в конечный момент расстояние между указателями будет n/2, то есть мы найдем середину списка.

3. **Удалить элемент из односвязного списка**

В решении используется дополнительный указатель `dummy_head` на `head`. Благодаря этому удаление как головы, так и внутренних элементов, происходит единообразно. Также используется указатель на предыдущий элемент `prev`. Старым указателем на начало `head` проходим по всем элементам списка и в случае совпадения значения узла с тем, что нужно удалить, меняем указатель `prev.next`, таким образом удаляя элемент из списка. В конце возвращаем `dummy_head.next`, который всегда будет содержать указатель на начало обновленного списка, даже если из него удалены все элементы.

4. **Является ли одна строка исходной для другой**

Для решения используются два указателя `pa` и `pb`. Первый идет по строке a, второй - по b, на каждом шаге сдвигается `pb`, а если элементы строк совпадают, то сдвигается и `pa`. Цикл заканчивается, когда один из указателей дойдет до конца своей строки. Если это указатель `pa`, значит, все элементы строки a содержатся в b, потому что указатель `pa` сдвигается только в том случае, когда он указывает на тот же самый элемент, что и `pb` в строке b.

5. **Метод двух указателей (поиск пары)**

Используются два указателя: один идет с начала массива, второй - с конца. На каждом этапе сдвигается один из указателей, либо цикл завершается, если условие выполнено и найдено искомое число. Этих сдвигов будет максимум n, где n - длина массива. Поэтому решение работает за $O(n)$.

6. **Является ли слово палиндромом?**

*(здесь реализация такая же, как и в задаче на LeetCode, то есть знаки препинания, пробелы и в целом какие-либо символы помимо букв и цифр на то, является ли слово палиндромом или нет, не влияют, то есть строка `"A man, a plan, a canal: Panama"` будет палиндромом).*

Используем 2 указателя, на каждом шаге сначала пропускаем все символы, которые не влияют на палиндромность, а потом сравниваем значения строки по каждому из указателей (приведя их к нижнему регистру). Каждый элемент строки просматривается максимум 1 раз, поэтому сложность линейная.

7. **Метод двух указателей (удаление дубликатов)**

Используются 2 указателя, один следит за позицией для записи нового элемента, второй идет по всем элементам массива, и в случае нахождения нового элемента (который еще не записан в начале массива) происходит перезапись, после чего первый указатель сдвигается. Корректность операций достигается за счет отсортированности оригинального массива.

8. **Слияние двух отсортированных списков**

На каждом шаге последовательно проверяем, какой из элементов списков меньше, и обрабатываем списки в соответствием с этим (переставляем указатели). При этом каждый элемент списков обрабатывается максимум 1 раз. Так как в решении меняются только указатели, а не создаются новые элементы списка, решение является $O(1)$ по памяти.
